# Личные настройки Claude Code

Этот файл содержит глобальные настройки для Claude Code, применяемые ко всем проектам.

## Язык общения

**ВАЖНО:** Всегда отвечай на русском языке, независимо от языка кода или технической документации.

## Стиль работы

- Будь лаконичным и конкретным
- Не используй эмодзи без явной просьбы
- Объясняй сложные вещи простым языком
- При изменении кода всегда показывай что именно меняется

## Предпочтения по коду

- Предпочитай чистоту и читаемость производительности (если нет явных требований)
- Используй современные подходы и best practices
- Добавляй комментарии только там, где код не очевиден
- Следуй существующему стилю кода в проекте

## Git коммиты

- Используй conventional commits где применимо
- Пиши сообщения коммитов на английском (это стандарт)
- Делай атомарные коммиты (одна логическая единица изменений)
- В описании коммита объясняй "почему", а не "что"
- **Не добавляй Co-Authored-By или другие attribution метаданные**

## Git ветки

- Именуй новые ветки по gitflow-avh конвенции: `feature/`, `bugfix/`, `hotfix/`, `release/`, `support/`
- Выбирай тип ветки исходя из контекста беседы
- Slug ветки также подбирай исходя из контекста беседы
